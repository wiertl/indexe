1) Instrukcja wyboru – składnia teoretyczna
if (warunek_logiczny1) { } if (warunek_logiczny2) { }

2) Instrukcja wyboru – schemat blokowy
Schemat blokowy przedstawia sprawdzenie warunku logicznego. Jeśli warunek jest spełniony, wykonywane są instrukcje. Jeśli nie – program przechodzi dalej.

3) Instrukcja alternatywy – składnia teoretyczna
if (warunek_logiczny) { } else { }

4) Instrukcja alternatywy – schemat blokowy
Schemat blokowy zawiera warunek logiczny oraz dwa wyjścia: TAK (prawda) – wykonywane są instrukcje z if, NIE (fałsz) – wykonywane są instrukcje z else.

5) Operatory porównania
== jest równe (2==3 → fałsz)
!= nie jest równe (2!=3 → prawda)
> jest większe (25>3 → prawda)
< jest mniejsze (2<3 → prawda)
>= większe lub równe (25>=3 → prawda)
<= mniejsze lub równe (2<=3 → prawda)

6) Uwaga do operatorów porównania
Należy odróżnić == (porównanie) od = (przypisanie).
== oznacza porównanie wartości,
= oznacza przypisanie wartości do zmiennej, np. a=5;

7) Operatory logiczne
&& (i) np. (x < 9 && y > 2) → prawda
|| (lub) np. (x==8 || y==6) → fałsz
! (zaprzeczenie) np. !(x==y) → prawda
